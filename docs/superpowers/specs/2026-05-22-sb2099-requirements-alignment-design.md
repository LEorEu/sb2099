# sb2099 需求对齐设计

> 日期：2026-05-22
> 状态：已完成口头确认，等待书面复核
> 关联文档：`设计文档.md`、`分析报告.md`、`api.md`

## 目标

`sb2099` 面向斗鱼 2099 房间观众，提供更纯粹的烂梗投稿、收藏、发现与单条发送体验。第一版保留投稿库和直播热门两个内容来源，但严格分层：

- 投稿库负责长期沉淀和油猴使用。
- 直播热门负责发现现场候选内容。
- 油猴只消费投稿库，不直接消费直播热门。
- 不做用户系统，不做 AI 造梗、赛事衍生玩法、贴吧、广告和自动连续发送。

## 公开产品结构

第一版主站只有三个核心公开页面。

### 首页

- 简短介绍项目用途。
- 明显展示油猴安装入口。
- 直接展示投稿表单：正文输入和固定 tag 多选。
- 展示随机一条投稿库烂梗，支持复制。
- 展示少量最新投稿预览，支持复制。

### 全部烂梗

- 作为正式投稿库页面。
- 默认按最新投稿排序。
- 支持关键词搜索。
- 支持多 tag 筛选；多个 tag 按满足任一匹配。
- 支持投稿按钮，打开与首页同构的投稿弹窗。
- 投稿库条目支持复制、本地主站收藏和“不合适”反馈。

### 热门弹幕

- 作为直播现场发现页，不承担正式投稿库语义。
- 公开页只展示严格榜单，初期形态为今日 Top 10、本周 Top 50。
- 榜单条目支持网页复制。
- 未入库条目提供小 `+` 入库入口。
- 已入库和未入库条目可用背景或边框区分。
- 桌面 hover 时，已入库条目展示绑定 tag；未入库条目展示入库提示。

## 投稿与沉淀

### 普通投稿

- 首页表单和全部烂梗投稿弹窗共用字段与校验。
- 必须选择至少一个管理员维护的固定 tag。
- 内容长度上下限由后台设置控制。
- tag 不开放给访客新增；新 tag 由粉丝群与管理员线下协商后后台加入。
- 公开投稿不展示投稿者信息。

### 热门提升投稿

- 热门页未入库条目点 `+` 后打开同一套投稿弹窗。
- 正文自动带入且只读。
- 用户只补 tag 后提交。
- 已入库热门条目不再显示入库入口。

### 去重与审核

- 用保守归一化后的正文去重。
- 重复投稿不新建，直接提示“投稿库已有相同内容”并展示已有条目。
- 普通内容直接公开。
- 命中投稿待审规则的普通投稿和热门提升投稿进入 `pending`。
- 管理员审核时可修正 tag，可通过或拒绝，不修改正文。
- 已公开投稿管理员可改 tag、可软删，不修改正文。

## 热门生成与过滤

直播热门以 2099 房间弹幕聚合生成。

- 归一化保持保守：清理首尾空白、合并空白、统一全半角、去零宽和不可见控制字符。
- 不主动剥离 emoji。
- 不大规模忽略标点。
- 不强行合并轻微改写。
- 公开阈值以时间窗内的不同发送者数为主，阈值后台可配置。
- 发送次数仍可用于排序、榜单展示和后台观察。

过滤规则拆成两套：

- 直播降噪规则只影响热门弹幕生成和曝光。
- 投稿待审规则只影响普通投稿和热门提升投稿。

管理员不承担全量热门整理工作。访客可从热门榜单补 tag 提升内容，管理员只做风险和质量兜底。

## 收藏、反馈与油猴

### 收藏

- 主站和油猴都支持本地收藏分组。
- 可创建、重命名、删除分组。
- 支持导入和导出，第一版不自动同步。
- 收藏只针对投稿库条目。
- 收藏数据初期只保存条目编号和分组关系，不保存正文快照。

### 反馈

- 正向热度主要使用复制数。
- 投稿库不做正负投票。
- 主站投稿库条目只提供单按钮“不合适”，给管理员公共负反馈信号。
- “不合适”不自动下架，后台按反馈次数和最近反馈处理。

### 油猴

- 只查询正式投稿库。
- 支持关键词搜索和 tag 筛选。
- 支持结果列表逐条复制和单条发送。
- 支持本地收藏分组。
- 支持导入主站收藏、导出油猴收藏。
- 支持本地屏蔽和恢复投稿库条目。
- 不展示直播热门，不做自动连续发送。

## 后台边界

后台第一版至少覆盖：

- 固定 tag 管理。
- 投稿长度、限流和阈值设置。
- 直播降噪规则管理和重算。
- 投稿待审规则管理。
- `pending` 投稿审核。
- “不合适”反馈处理。
- 软删投稿恢复或永久删除。
- 热门聚合与过滤排查。

## 设计取舍

本设计保留直播热门，是为了帮助投稿库冷启动并发现新梗；同时把油猴限制在正式投稿库上，避免直播噪声直接进入直播间发送工具。收藏导入导出替代账号同步，先验证个性化梗单的价值，再决定后续是否改进同步体验。
